## 二、七大核心技能诉求

### 2.1 提示工程（Prompt Engineering）

**定位**：与模型沟通的底层语言，LLM原生应用的"编程语言"。

**为什么关键**：设计不良的Prompt会导致输出不稳定、幻觉频发、成本失控等一系列生产环境问题。

**核心能力要求**：

| 能力维度 | 具体要求 | 重要程度 |
|---------|---------|---------|
| **角色设定** | 明确告诉模型它是谁、有什么能力边界 | ⭐⭐⭐⭐⭐ |
| **上下文注入** | 将任务相关的背景信息以结构化方式传入 | ⭐⭐⭐⭐⭐ |
| **输出格式约束** | 通过JSON Schema或示例强制规范输出结构 | ⭐⭐⭐⭐⭐ |
| **链式思维（CoT）** | 引导模型分步推理，提升复杂任务准确率 | ⭐⭐⭐⭐⭐ |
| **少样本学习（Few-Shot）** | 通过高质量示例快速适配特定任务风格 | ⭐⭐⭐⭐ |
| **提示版本管理** | A/B测试评估和自动化优化流程 | ⭐⭐⭐⭐ |

**2026年进阶要求**：
- 异步并发编程中的Prompt管理
- 多Agent协同Prompt设计
- 目标拆解式Prompt与反思迭代循环
- 自我批判与重试循环

### 2.2 RAG（检索增强生成）

**定位**：让AI真正"知道"你的数据，解决模型知识时效性和私有数据访问的根本性限制。

**技术链路要求**：

```
文档解析与分块 → 向量化（Embedding）→ 向量数据库 → 相似度检索 → 答案生成与溯源
```

**核心能力层次**：

| 层次 | 能力 | 技术实现 |
|-----|------|---------|
| **基础RAG** | 简单检索+生成 | 向量嵌入、基础向量数据库 |
| **高级RAG** | 查询重写、结果重排 | 混合检索、Re-ranking |
| **Agent化RAG** | 自适应检索策略 | Agent RAG、多路召回 |
| **图谱增强RAG** | 知识图谱融合 | Graph RAG、结构化知识 |
| **多模态RAG** | 图文音视频检索 | 多模态Embedding、跨模态检索 |

**2026年关键变化**：
- Agent RAG、自适应RAG已成为标配技术
- 多模态数据检索和知识图谱融合成为主流
- 检索质量成为决定Agent输出质量的关键瓶颈

**关联架构**：质量智能体的 [Shared Memory Layer](file:///Users/bohaiqing/opensource/git/ai-testing/docs/QUALITY_AGENT_ARCHITECTURE.md)（VectorDB + Knowledge Graph）正是RAG的核心基础设施。

### 2.3 工具调用（Tool Use）

**定位**：赋予AI行动能力，Skill是Agent的"手"，没有Skill，Agent只能思考，无法行动。

**核心工具类型**：

| 工具类型 | 示例 | 应用场景 | 重要程度 |
|---------|------|---------|---------|
| **搜索工具** | 搜索引擎、企业内部检索 | 实时信息获取 | ⭐⭐⭐⭐⭐ |
| **计算工具** | Python代码执行、计算器 | 精确计算、数据分析 | ⭐⭐⭐⭐⭐ |
| **数据库工具** | SQL查询、NoSQL操作 | 数据存取、业务查询 | ⭐⭐⭐⭐⭐ |
| **浏览器自动化** | Selenium、Playwright | Web操作、数据采集 | ⭐⭐⭐⭐ |
| **企业API集成** | RESTful SDK、内部系统 | 业务系统对接 | ⭐⭐⭐⭐ |
| **代码执行** | 沙箱执行、代码解释器 | 自动化处理、数据分析 | ⭐⭐⭐⭐ |

**2026年关键变化**：
- 本地LLM的成熟让开发者无需依赖云端API也能构建高性能Agent
- 工具能力自动发现与适配成为新趋势
- 工具权限管控和安全沙箱执行成为必选项

**技术实现要点**：
- 自定义工具开发
- 预置工具使用
- 工具调用标准协议（Function Calling）
- 工具权限与安全管理

### 2.5 记忆与状态管理

**定位**：构建"长记忆"Agent，保持对话连贯性、沉淀执行经验。

**记忆层次模型**：

| 层次 | 能力 | 技术实现 | 应用场景 |
|-----|------|---------|---------|
| **短时记忆** | 会话状态维护 | Redis、SQLite | 单轮/多轮对话 |

**核心技能要求**：
- 长短时记忆区分与管理
- 会话状态维护与恢复
- 向量记忆库构建与检索
- 经验复盘与知识沉淀机制

### 2.6 安全与治理

**定位**：生产级AI应用的底线，防止敏感数据泄露、权限提升和系统滥用。

**基于STRIDE威胁模型的安全要求**：

| 威胁类别 | 具体风险 | 缓解措施 | 验证方式 |
|---------|---------|---------|---------|
| **S**poofing（欺骗） | 伪造Agent身份调用工具或API | mTLS双向认证 + JWT签名验证；API Key定期轮换 | 渗透测试：尝试伪造Agent身份 |
| **T**ampering（篡改） | 篡改RAG知识库中的测试模式或规则 | HMAC签名校验；WORM存储 | 完整性校验测试 |
| **R**epudiation（否认） | 危险操作无证据 | 不可篡改日志；区块链审计链 | 日志审计测试 |
| **I**nformation Disclosure（信息泄露） | LLM输出包含生产敏感数据 | 输出层DLP扫描；敏感数据正则匹配拦截 | 敏感数据注入测试 |
| **D**enial of Service（拒绝服务） | 超大文件导致解析死循环 | 输入大小限制；超时熔断 | 模糊测试 |
| **E**levation of Privilege（权限提升） | 沙箱逃逸获取宿主机权限 | gVisor系统调用拦截 + seccomp | 容器逃逸测试 |

**数据安全分级处理**：

| 敏感级别 | 数据类型 | 示例 | 处理策略 |
|---------|---------|------|---------|
| **Critical** | 生产凭证、私钥、数据库密码 | `DB_PASSWORD=xxx` | **禁止进入系统**：预提交钩子拦截，构建失败 |
| **High** | API Token、Session ID、内部IP | `Authorization: Bearer xxx` | **强制脱敏**：替换为占位符，进入系统前扫描 |
| **Medium** | 业务敏感数据（手机号、银行卡号） | `138****1234` | **按需脱敏**：根据场景决定是否脱敏 |
| **Low** | 内部服务名、非敏感配置 | `payment-service` | **允许通过**：正常处理 |

**AI置信度驱动的自动化分级**（Harness 2026）：

| 置信度范围 | 风险等级 | 执行策略 |
|-----------|---------|---------|
| ≥ 0.95 | 低 | 自动执行 |
| 0.85 ~ 0.95 | 中 | 通知后执行（5分钟撤销窗口） |
| < 0.85 | 高 | 强制确认 |
| 任意（关键操作） | 关键 | 强制确认（生产部署等） |

### 2.7 评估与优化

**定位**：AI系统的质量保障与传统软件完全不同，需要新的评估方法论。

**基于五维能力模型的评估体系**：

| 能力维度 | 评估重点 | 技术方法 | 关键指标 |
|---------|---------|---------|---------|
| **感知层** | 质量指标计算准确性 | 数据验证、指标监控 | 数据采集完整率、指标准确率 |
| **认知层** | 风险识别准确率 | A/B测试、人工评估 | 风险预测准确率、误报率 |
| **决策层** | 策略生成合理性 | 对抗式验证、Self-Play | 策略采纳率、效果提升度 |
| **执行层** | 测试执行稳定性 | 沙箱执行、超时熔断 | 执行成功率、平均耗时 |
| **进化层** | 策略优化效果 | 强化学习、经验沉淀 | 策略迭代效率、知识复用率 |

**2026年新增评估要求**：
- **Self-Play验证机制**：对抗式验证测试代码质量
- **AI置信度评估**：每个决策点的置信度计算与分级
- **多模态质量评估**：图像/语音/视频生成质量评估
- **公平性与偏见检测**：模型输出的公平性审计

## 三、技术栈推荐（2026年）

基于 [MULTI_LANGUAGE_AGENT_ARCHITECTURE_ASSESSMENT.md](file:///Users/bohaiqing/opensource/git/ai-testing/docs/MULTI_LANGUAGE_AGENT_ARCHITECTURE_ASSESSMENT.md) 的评估结论：

> **坚持用Python单语言开发，至少在项目前12-18个月内不要引入第二语言。**

| 层次 | 推荐技术 | 说明 |
|-----|---------|------|
| **编程语言** | Python 3.10+ | AI生态最完善，开发效率高 |
| **Agent框架** | LangChain / LangGraph / AutoGen / CrewAI | 主流框架，社区活跃 |
| **向量数据库** | Milvus / Pinecone / Weaviate / Chroma | 知识检索基础设施 |
| **LLM接口** | OpenAI API / DeepSeek / 本地LLM(Ollama) | 云端+本地混合部署 |
| **记忆存储** | Redis / SQLite / PostgreSQL | 短时记忆与会话状态 |
| **部署** | Docker + K8s / 云原生 | 生产级部署 |
| **监控** | Prometheus + Loki / Grafana | 可观测性与日志 |
| **沙箱执行** | gVisor + seccomp | 代码执行安全 |

**业界共识**：

| 公司/项目 | 语言选择 | 规模 |
|-----------|---------|------|
| AutoGPT | Python | 150k+ stars |
| MetaGPT | Python | 40k+ stars |
| CrewAI | Python | 20k+ stars |
| LangGraph | Python | 10k+ stars |
| OpenAI Assistants | Python SDK | 企业级 |
