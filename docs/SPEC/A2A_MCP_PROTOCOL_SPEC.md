## 核心设计决策

> 采用 **A2A（Agent-to-Agent）协议** 作为 Agent 间统一通信标准，**MCP（Model Context Protocol）** 作为工具调用标准，形成双协议协同架构。

### .1 双协议协同架构


### .2 A2A 协议（Agent间通信）

| A2A核心能力 | 说明 | 在质量中台中的应用 |
|-------------|------|-------------------|
| **Agent Card** | 身份声明与能力发现 | 每个服务发布自身能力描述 |
| **JSON-RPC 2.0** | 消息传递格式 | 服务间请求/响应标准化 |
| **Task状态机** | 任务生命周期管理 | 任务创建→进行中→完成/失败 |
| **SSE流式响应** | 服务器推送事件 | 实时任务状态推送 |
| **Webhook通知** | 异步回调通知 | 长任务完成通知 |

**A2A协议链接**：
- 官网：https://www.a2aprotocol.net/
- 规范：https://a2a-protocol.org/v0.2.6/specification/

### .3 MCP 协议（工具调用标准）

> **核心设计决策**：MCP 作为 Agent 与外部工具/服务通信的统一标准，解决工具发现、调用、安全隔离问题。

| MCP核心能力 | 说明 | 在质量中台中的应用 |
|-------------|------|-------------------|
| **Tool Registry** | 工具注册与发现 | 统一注册监控、日志、数据库等工具 |
| **Capability Request** | 能力请求/协商 | Agent动态发现可用工具能力 |
| **Tool Schema** | 工具输入输出定义 | 标准化参数传递与结果解析 |
| **Context Exchange** | 上下文传递 | 工具执行上下文共享 |

**MCP协议链接**：
- 官网：https://modelcontextprotocol.io/
- 规范：https://spec.modelcontextprotocol.io/

### .4 A2A vs MCP 职责边界

| 场景 | 协议选择 | 原因 |
|------|---------|------|
| Agent 间任务分发 | **A2A** | 需要完整的Agent能力发现与状态管理 |
| Agent 调用外部工具 | **MCP** | 标准化工具接口，支持动态发现 |
| Agent 获取系统信息 | **MCP** | 数据库、K8s、日志等通过MCP服务暴露 |
| 跨平台Agent协作 | **A2A** | 支持多平台、多语言的Agent互操作 |
| IDE/编辑器集成 | **MCP** | CodeBuddy、Claude Desktop等原生支持 |

**协同示例**：
```
运维Agent (A2A) ──▶ 根因分析任务分发 ──▶ 日志分析Agent (A2A)
                              │
                              ▼ (MCP)
                    ┌─────────────────┐
                    │ 日志查询服务    │
                    │ (MCP Server)    │
                    └─────────────────┘
```
