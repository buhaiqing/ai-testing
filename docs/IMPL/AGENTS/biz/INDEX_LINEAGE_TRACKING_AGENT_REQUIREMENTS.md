## 1. 概述

### 1.1 Agent定位

指标血缘追踪Agent是质量智能中台运营场景的核心Agent之一，负责自动梳理指标来源和计算逻辑，帮助运营团队快速定位数据问题源头，解决指标口径不统一的痛点。

### 1.2 核心价值

| 价值维度 | 描述 |
|---------|------|
| **问题溯源** | 快速定位指标异常的根本原因，从小时级缩短到分钟级 |
| **口径统一** | 消除不同系统间指标口径差异，确保数据一致性 |
| **变更可控** | 评估上游变更影响，提前规避潜在问题 |
| **知识传承** | 沉淀指标定义和计算逻辑，降低团队知识门槛 |

## 3. 功能需求详情

### 3.1 血缘解析模块

#### 3.1.1 SQL血缘解析

| 需求ID | 需求描述 | 优先级 | 验收标准 |
|--------|---------|-------|---------|
| LINEAGE-001 | 支持解析主流SQL语法（MySQL、PostgreSQL、ClickHouse等） | P0 | 能正确解析SELECT、JOIN、聚合、子查询等 |
| LINEAGE-001 | 自动识别字段来源表和字段 | P0 | 输出字段级别的血缘关系 |
| LINEAGE-001 | 支持CTE（Common Table Expression）解析 | P1 | 能处理WITH语句定义的临时表 |
| LINEAGE-001 | 支持嵌套子查询解析 | P1 | 能追踪3层以上嵌套子查询 |
| LINEAGE-001 | 支持UDF（用户自定义函数）血缘追踪 | P2 | 能识别函数输入输出字段 |

#### 3.1.2 ETL血缘解析

| 需求ID | 需求描述 | 优先级 | 验收标准 |
|--------|---------|-------|---------|
| LINEAGE-002 | 支持解析DataX配置文件 | P0 | 能识别源表、目标表、字段映射 |
| LINEAGE-002 | 支持解析Apache DolphinScheduler任务 | P0 | 能识别任务依赖关系 |
| LINEAGE-002 | 支持解析Airflow DAG定义 | P1 | 能识别DAG中的任务依赖 |
| LINEAGE-002 | 支持解析Kettle转换 | P1 | 能识别输入输出步骤 |

#### 3.1.3 配置血缘解析

| 需求ID | 需求描述 | 优先级 | 验收标准 |

| LINEAGE-003 | 支持解析指标配置（阈值、计算公式） | P0 | 能识别指标依赖的上游字段 |
| LINEAGE-003 | 支持解析报表配置 | P1 | 能识别报表依赖的指标 |
| LINEAGE-003 | 支持解析告警规则配置 | P1 | 能识别告警依赖的数据源 |

## 4. 接口设计

### 4.1 核心API

| 接口名称 | 方法 | 路径 | 功能描述 |
|---------|------|------|---------|
| 血缘解析 | POST | /api/v1/lineage/parse | 解析SQL/ETL获取血缘 |
| 血缘查询 | GET | /api/v1/lineage/{id} | 查询指定节点的血缘 |
| 影响分析 | POST | /api/v1/lineage/impact | 分析变更影响范围 |
| 口径对比 | POST | /api/v1/lineage/compare | 对比两个指标的口径 |
| 异常溯源 | POST | /api/v1/lineage/trace | 追溯异常指标源头 |
| 血缘图谱 | GET | /api/v1/lineage/graph | 获取图谱数据 |
