## 核心术语

| 术语 | 英文对照 | 定义 | 使用场景 |
|-----|---------|------|---------|
| **AI模型** | AI Model | 泛指人工智能模型，包括大语言模型、视觉模型、多模态模型等 | 替代"LLM"等具体技术术语，用于架构描述和决策讨论 |
| **知识检索** | Knowledge Retrieval | 基于向量的语义检索与增强生成技术 | 替代"RAG"，用于描述知识库查询能力 |
| **端到端测试** | End-to-End Testing | 从用户视角验证完整业务流程的测试方法 | 替代"E2E"，用于测试策略讨论 |
| **视觉识别** | Visual Recognition | 基于多模态模型的图像识别与理解能力 | 替代"Vision Model"，用于UI测试和元素定位 |
| **代码检查** | Code Linting | 静态代码质量检查与规范验证 | 替代"Lint"，用于代码质量保障 |
| **流式响应** | Streaming Response | 分块逐步返回AI生成内容的技术 | 用于优化用户体验，降低感知延迟 |
| **多路召回** | Multi-Channel Retrieval | 同时使用多种检索策略（向量、关键词、混合）提升准确率 | 用于知识检索优化 |
| **超时熔断** | Timeout Circuit Breaker | 任务执行超过阈值自动终止的保护机制 | 用于Agent执行稳定性保障 |
| **增量更新** | Incremental Update | 仅更新变化部分而非全量刷新的数据同步策略 | 用于知识库和规则库维护 |
| **沙箱执行** | Sandbox Execution | 隔离环境中运行代码的安全机制 | 用于测试代码执行安全 |

## 架构术语

| 术语 | 英文对照 | 定义 |
|-----|---------|------|
| **协调器** | Orchestrator | 质量协调智能体，负责任务分解、资源调度、策略决策 |
| **专业Agent** | Specialist Agent | 具备特定领域能力的智能体，如数据采集、风险评估、测试执行 |
| **扩展Agent** | Extension Agent | 按需加载的辅助智能体，如策略规划、根因诊断、学习进化 |
| **共享记忆层** | Shared Memory Layer | 多Agent共享的知识存储与检索层，包含向量存储、规则引擎、知识图谱 |
| **感知层** | Perception Layer | 负责从外部系统采集质量数据的层次 |
| **认知层** | Cognition Layer | 负责质量状态评估、风险识别、根因分析的层次 |
| **决策层** | Decision Layer | 负责生成质量策略、资源分配决策的层次 |
| **执行层** | Execution Layer | 负责测试执行、结果分析、报告生成的层次 |
| **进化层** | Evolution Layer | 负责经验沉淀、知识更新、策略优化的层次 |

## 能力等级术语

| 术语 | 定义 |
|-----|------|
| **Level 1: 数据感知** | 能够采集、清洗、计算质量指标的基础能力 |
| **Level 2: 认知分析** | 能够评估质量状态、识别风险、分析根因的能力 |
| **Level 3: 决策规划** | 能够生成策略、优化方案、协调执行的能力 |
| **Level 4: 自主决策** | 在人工监督下独立完成质量决策的能力 |
| **Level 5: 自我进化** | 能够自主学习、自动优化、零人工干预的能力 |

## 术语使用规范

### AGENTS.md 使用规范
- 必须使用抽象术语（如"AI模型"、"知识检索"）
- 禁止出现具体技术产品名称（如"GPT-4"、"Milvus"、"Redis"）
- 首次出现术语需链接到本文档

### docs/ 技术文档使用规范
- 可使用具体技术术语，但**首次出现**需标注对应抽象术语
- 示例："使用向量数据库（知识检索的具体实现）存储语义向量"
- 必须在文档头部引用本文档

### 新增术语流程
1. 在本文档"变更历史"中登记新增术语
2. 更新术语表内容
3. 同步更新引用本文档的所有文档
4. 提交变更说明
