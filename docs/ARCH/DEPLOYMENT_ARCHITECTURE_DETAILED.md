## 核心设计决策

> Agent Runtime需支持Kubernetes与独立Docker两种部署形态，通过统一部署抽象层屏蔽底层差异，确保核心功能一致性。

---

## 部署形态特性对比

| 维度 | Kubernetes部署模式 | 独立Docker部署模式 |
|-----|------------------|------------------|
| **应用场景** | 需要弹性伸缩、高可用的生产环境 | 线下IDC环境 |
| **资源调度** | 自动化的动态资源分配 | 手动配置 |
| **服务编排** | Service/Ingress/DNS原生支持 | Docker网络+静态配置 |
| **扩缩容** | HPA自动扩缩 | 基于监控的手动扩缩 |
| **高可用** | 自动故障恢复、自愈 | 冗余部署+手动恢复 |
| **监控集成** | Prometheus/Grafana原生集成 | 轻量级监控组件 |
| **运维复杂度** | 中高（需K8S知识） | 低（纯Docker） |

## 资源管理策略

| 维度 | K8S模式 | Docker模式 |
|-----|--------|-----------|
| **资源配置** | ResourceQuota + LimitRange | 宿主机监控+静态分配 |
| **动态调整** | Vertical Pod Autoscaler (VPA) | 基于监控的手动调整 |
| **阈值告警** | Prometheus AlertManager | 轻量级exporter + 告警 |
| **优化策略** | 基于历史数据的智能调度 | 基于阈值的简单扩缩容 |

---

## 网络与服务发现

| 维度 | K8S模式 | Docker模式 |
|-----|--------|-----------|
| **服务注册** | K8S Service + DNS | Docker网络+Consul |
| **负载均衡** | K8S Service内部负载 | Docker网络轮询 |
| **外部访问** | Ingress Controller | Docker端口映射 |
| **服务发现** | K8S DNS + 环境变量 | 静态配置文件 |

---

## 扩缩容与高可用

| 维度 | K8S模式 | Docker模式 |
|-----|--------|-----------|
| **自动扩缩容** | HPA + 自定义metrics | 手动扩缩容工具 |
| **故障检测** | K8S健康检查+Liveness | Docker HEALTHCHECK |
| **故障恢复** | 自动重启+拉起 | 监控脚本+手动 |
| **数据持久化** | PersistentVolume | Docker Volume |

---

## 监控告警体系

| 维度 | K8S模式 | Docker模式 |
|-----|--------|-----------|
| **指标采集** | cAdvisor + kube-state-metrics | node-exporter + cAdvisor |
| **日志采集** | Fluentd/Fluent-bit → Loki | Filebeat → Loki |
| **链路追踪** | OpenTelemetry Operator | OpenTelemetry Collector |
| **告警规则** | PrometheusRule CRD | 静态PrometheusRule |
| **可视化** | Grafana Dashboards | 共享Grafana实例 |

## 技术取舍原则

```
┌─────────────────────────────────────────────────────────────────┐
│                   技术取舍原则                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 通用核心优先                                                 │
│     └── 跨环境通用功能：编排引擎、状态管理、三层协作架构          │
│                                                                 │
│  2. 环境特定增强                                                 │
│     └── K8S：自动扩缩、自愈、智能调度                            │
│     └── Docker：轻量监控、简化运维                               │
│                                                                 │
│  3. 功能裁剪标准                                                 │
│     └── 独立Docker模式不支持：HPA、VPA、Operator                 │
│     └── 提供等效替代：手动扩缩容脚本、简化管理界面               │
│                                                                 │
│  4. 性能目标                                                     │
│     └── K8S模式：启动时间<30s，扩缩容<60s                       │
│     └── Docker模式：启动时间<15s，配置同步<10s                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 部署形态自动检测

```yaml
# 部署形态检测逻辑
deployment_detection:
  k8s_indicators:
    - env_var: "KUBERNETES_SERVICE_HOST"
    - file: "/var/run/secrets/kubernetes.io/serviceaccount/token"
    - api: "/apis/apps/v1/namespaces"

  docker_indicators:
    - env_var: "DOCKER_HOST"
    - file: "/.dockerenv"
    - cmd: "docker info"

  fallback: "docker"  # 默认使用Docker模式
```
