## 一、多 Agent 协作架构设计

### 1.1 架构演进概述

本文档详细阐述质量智能体的核心架构组件，与AGENTS.md v4.0保持一致：

| 组件 | 解决的问题 | 核心技术 | 成熟度 |
|-----|-----------|---------|--------|
| **三层协作架构** | Agent间同步/异步通信、知识共享 | 函数调用+消息总线+知识共享 | 业界成熟 |
| **平台核心基础设施** | Agent生命周期管理、消息路由、状态管理、安全、可观测性 | 分布式架构 | 业界成熟 |
| **CQRS + Event Sourcing** | 状态管理、审计追溯 | 事件驱动架构 | 业界成熟 |
| **Self-Play 验证机制** | 测试代码质量验证 | 对抗式验证 | 创新实践 |

> ⭐ **架构决策**：根据AGENTS.md v4.0的决策，Agent间通信采用三层协作架构（函数调用/消息总线/知识共享）。

### 1.2 三层协作架构概述

三层协作架构分别解决不同问题，互为补充：

| 层级 | 名称 | 解决问题 | 技术实现 | 延迟 |
|-----|------|---------|---------|------|
| **Layer1** | 函数调用 | 下一步该调用哪个Agent | @tool / Python直接调用 | < 10ms |
| **Layer2** | 消息总线 | 如何通知多个Agent | Kafka / Redis PubSub | 10-100ms |
| **Layer3** | 知识共享 | 如何共享状态和知识 | Redis + Milvus + Neo4j | < 5ms |

**三层架构职责边界**：
- **Layer1 函数调用**：Orchestrator同步编排Agent、固定流程、实时响应
- **Layer2 消息总线**：异步通知、一对多广播、耗时任务解耦
- **Layer3 知识共享**：RAG增强、历史案例共享、用户上下文传递

## 八、Knowledge Graph 工程实现

### 8.1 知识图谱核心实体

| 实体类型 | 属性 | 关系 |
|---------|------|------|
| **CodeModule** | name, language, path, owner | contains, depends_on, tested_by |
| **TestCase** | name, type, coverage, status | tests, covers, related_to |
| **QualityIssue** | severity, type, status, assignee | found_in, fixed_by, similar_to |
| **Developer** | team, role, expertise | owns, contributed_to, responsible_for |

### 8.2 知识图谱查询模式

```cypher
// 查询模块的测试覆盖情况
MATCH (m:CodeModule {name: $module})
OPTIONAL MATCH (m)-[:tested_by]->(t:TestCase)
RETURN m.name, count(t) as test_count, collect(t.name) as tests

// 查询相似质量问题
MATCH (i1:QualityIssue {id: $issue_id})-[:similar_to]->(i2:QualityIssue)
RETURN i2.id, i2.description, i2.solution
```

## 九、容错降级策略

### 9.1 故障分级与响应

| 故障级别 | 触发条件 | 响应策略 | RTO |
|---------|---------|---------|-----|
| **P0-关键** | 核心服务完全不可用 | 立即切换备用 + 告警 | < 5min |
| **P1-严重** | 主要功能受损 | 降级模式 + 人工介入 | < 15min |
| **P2-一般** | 次要功能异常 | 日志记录 + 定时处理 | < 1h |
| **P3-轻微** | 性能下降 | 监控观察 | < 4h |

### 9.2 典型故障场景处理

| 故障场景 | 降级策略 | 恢复机制 |
|---------|---------|---------|
| **LLM服务不可用** | 切换备用模型/缓存结果 | 自动重试 + 告警 |
| **VectorDB不可用** | 降级为关键词搜索 | 异步重建索引 |
| **Token预算超支** | 限制非关键请求 | 配额重置后恢复 |
| **Agent实例故障** | 流量切换到健康实例 | 自动重启故障实例 |
